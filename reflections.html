July 5th, 2017---Reflections on MVC Paradigm and Web Framework

Questions : "In your own words, what is the MVC Paradigm? How is it used?"
"What is the difference between a framework and a library?"
"Why might you use a framework instead of creating an app yourself?"

A MVC Paradigm is a pattern for relating user interface to data models and is divided into three parts. The difference between a framework and a library is that a library is a set of functions used for one purpose, while a framework is a collection or set of those libraries. I might use a framework instead of creating an app myself because it would be faster and more efficient because the framwork already provides me a template to make an application, so that I won't have to start from scratch.
